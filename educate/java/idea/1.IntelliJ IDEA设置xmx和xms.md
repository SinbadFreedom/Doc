Intelij IDEA在跑大内存的程序时.会出现outofmemory异常,解决办法是设置-Xms和-Xmx参数.

1 打开`Intelij IDEA`的安装目录.
===
```
D:\Program Files\JetBrains\WebStorm 2017.1.4\bin
```

2 修改webstorm64.exe.vmoptions文件
===
```
-Xms128m
-Xmx750m
-XX:ReservedCodeCacheSize=240m
-XX:+UseConcMarkSweepGC
-XX:SoftRefLRUPolicyMSPerMB=50
-ea
-Dsun.io.useCanonCaches=false
-Djava.net.preferIPv4Stack=true
-XX:+HeapDumpOnOutOfMemoryError
-XX:-OmitStackTraceInFastThrow
```
修改里面的xms和xmx参数,这两个参数是设置java程序的最大和最小内存.
```
-Xms1024m
-Xmx2048m
```
如果是32位Intelij IDEA就修改webstorm.exe.vmoptions文件中的参数.
