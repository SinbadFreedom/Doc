2.1 一个简短的HTML 2.1 CSS教程
本节不具规范性。

在本教程中，我们展示了设计简单样式表有多容易。在本教程中，您需要了解一些HTML（请参阅[HTML4]）和一些基本的桌面出版术语。

我们从一个小的HTML文档开始：

<！DOCTYPE HTML PUBLIC“ -  // W3C // DTD HTML 4.01 // EN”>
<HTML>
  <HEAD>
  巴特的主页</ TITLE>
  </ HEAD>
  <BODY>
    <H1>巴赫的主页</ H1>
    约翰塞巴斯蒂安巴赫是一位多产的作曲家。
  </ BODY>
</ HTML>
要将H1元素的文本颜色设置为红色，您可以编写以下CSS规则：

  h1 {color：red}
CSS规则由两个主要部分组成：选择器（'h1'）和声明（'颜色：红色'）。在HTML中，元素名称不区分大小写，所以'h1'和'H1'一样。声明包含两部分：属性名称（'color'）和属性值（'red'）。虽然上面的例子试图只影响呈现HTML文档所需的其中一个属性，但是它自己有资格作为样式表。结合其他样式表（CSS的一个基本特征是样式表被组合），该规则将确定文档的最终呈现。

HTML 4规范定义了如何为HTML文档指定样式表规则：或者在HTML文档中，或者通过外部样式表。要将样式表放入文档中，请使用STYLE元素：

<！DOCTYPE HTML PUBLIC“ -  // W3C // DTD HTML 4.01 // EN”>
<HTML>
  <HEAD>
  巴特的主页</ TITLE>
  <STYLE type =“text / css”>
    h1 {color：red}
  </ STYLE>
  </ HEAD>
  <BODY>
    <H1>巴赫的主页</ H1>
    约翰塞巴斯蒂安巴赫是一位多产的作曲家。
  </ BODY>
</ HTML>
为了获得最大的灵活性，我们建议作者指定外部样式表; 它们可能会在不修改源HTML文档的情况下更改，并且可能会在多个文档中共享。要链接到外部样式表，可以使用LINK元素：

<！DOCTYPE HTML PUBLIC“ -  // W3C // DTD HTML 4.01 // EN”>
<HTML>
  <HEAD>
  巴特的主页</ TITLE>
  <LINK rel =“stylesheet”href =“bach.css”type =“text / css”>
  </ HEAD>
  <BODY>
    <H1>巴赫的主页</ H1>
    约翰塞巴斯蒂安巴赫是一位多产的作曲家。
  </ BODY>
</ HTML>
LINK元素指定：

链接的类型：一个“样式表”。
样式表通过“href”属性的位置。
被链接的样式表类型：“text / css”。
为了显示样式表和结构化标记之间的紧密关系，我们继续使用本教程中的STYLE元素。让我们添加更多颜色：

<！DOCTYPE HTML PUBLIC“ -  // W3C // DTD HTML 4.01 // EN”>
<HTML>
  <HEAD>
  巴特的主页</ TITLE>
  <STYLE type =“text / css”>
    身体{颜色：黑色; 背景：白色}
    h1 {color：red; 背景：白色}
  </ STYLE>
  </ HEAD>
  <BODY>
    <H1>巴赫的主页</ H1>
    约翰塞巴斯蒂安巴赫是一位多产的作曲家。
  </ BODY>
</ HTML>
样式表现在包含四条规则：前两个设置BODY元素的颜色和背景（最好将文本颜色和背景颜色设置在一起），而后两个则设置H1元素的颜色和背景。由于没有为P元素指定颜色，因此它将继承其父元素（即BODY）的颜色。H1元素也是BODY的子元素，但第二个规则会覆盖继承的值。在CSS中，不同值之间通常存在这种冲突，本规范描述了如何解决它们。

CSS 2.1拥有超过90个属性，包括'颜色'。我们来看看其他一些：

<！DOCTYPE HTML PUBLIC“ -  // W3C // DTD HTML 4.01 // EN”>
<HTML>
  <HEAD>
  巴特的主页</ TITLE>
  <STYLE type =“text / css”>
    身体 { 
      font-family：“Gill Sans”，sans-serif;
      font-size：12pt;
      保证金：3em; 
    }
  </ STYLE>
  </ HEAD>
  <BODY>
    <H1>巴赫的主页</ H1>
    约翰塞巴斯蒂安巴赫是一位多产的作曲家。
  </ BODY>
</ HTML>
首先要注意的是，几个声明被分组在花括号（{...}）所包围的块中，并以分号分隔，尽管最后一个声明后面也可以跟一个分号。

BODY元素的第一个声明将字体家族设置为“Gill Sans”。如果该字体不可用，则用户代理（通常称为“浏览器”）将使用“sans-serif”字体系列，该系列是所有用户代理所知的五种通用字体系列之一。BODY的子元素将继承'font-family'属性的值。

第二个声明将BODY元素的字体大小设置为12个点。“点”单位通常用于基于印刷的排版以指示字体大小和其他长度值。这是一个绝对单位的例子，它不相对于环境进行缩放。

第三个声明使用一个相对于周围环境的比例单位。“em”单位指的是元素的字体大小。在这种情况下，结果是BODY元素周围的边距比字体大三倍。

2.2 简要的XML 2.1教程
本节不具规范性。

CSS可以用于任何结构化文档格式，例如可扩展标记语言[XML10]的应用程序。实际上，XML比HTML更依赖于样式表，因为作者可以自己编写用户代理不知道如何显示的元素。

这是一个简单的XML片段：

<ARTICLE>
  弗雷德里克大帝遇见巴赫</ HEADLINE>
  <作者>约翰尼古拉斯福克尔</ AUTHOR>
  <PARA>
    有一天晚上，就像他得到他的时候一样 
    <INSTRUMENT> flute </ INSTRUMENT>准备好了和他的
    音乐家聚集在一起，一名军官给他列了一张名单
    已经抵达的陌生人。
  </ PARA>
</ ARTICLE>
为了以类似文档的方式显示这个片段，我们必须首先声明哪些元素是内联级别的（即不会导致换行符），哪些是块级别的（即导致换行符）。

INSTRUMENT {display：inline}
ARTICLE，HEADLINE，AUTHOR，PARA {display：block}
第一条规则声明INSTRUMENT为内联，第二条规则以逗号分隔的选择器列表声明所有其他元素为块级别。XML中的元素名称区分大小写，所以用小写字母（例如'instrument'）写的选择符不同于大写字母（例如'INSTRUMENT'）。

将样式表链接到XML文档的一种方法是使用处理指令：

<？xml-stylesheet type =“text / css”href =“bach.css”？>
<ARTICLE>
  弗雷德里克大帝遇见巴赫</ HEADLINE>
  <作者>约翰尼古拉斯福克尔</ AUTHOR>
  <PARA>
    有一天晚上，就像他得到他的时候一样 
    <INSTRUMENT> flute </ INSTRUMENT>准备好了和他的
    音乐家聚集在一起，一名军官给他列了一张名单
    已经抵达的陌生人。
  </ PARA>
</ ARTICLE>
可视用户代理可以将上述示例的格式设置为：

示例渲染   [d]

请注意，“长笛”一词仍然在段落内，因为它是内联元素INSTRUMENT的内容。

尽管如此，文本并未按照您所期望的方式格式化。例如，标题字体大小应该比文本的其余部分大，并且您可能希望以斜体显示作者的名称：

INSTRUMENT {display：inline}
ARTICLE，HEADLINE，AUTHOR，PARA {display：block}
HEADLINE {font-size：1.3em}
作者{font-style：italic}
文章，标题，作者，参与者{margin：0.5em}
可视用户代理可以将上述示例的格式设置为：

示例渲染   [d]

向样式表中添加更多规则将允许您进一步描述文档的表示。

2.3 CSS 2.1处理模型
本部分直至但不包括其小节都是非规范性的。

本节介绍支持CSS的用户代理如何工作的一种可能模式。这只是一个概念模型; 真正的实现可能有所不同

在此模型中，用户代理通过执行以下步骤来处理源代码：

解析源文档并创建文档树。
确定目标媒体类型。
检索与为目标媒体类型指定的文档关联的所有样式表。
通过为适用于目标媒体类型的每个属性分配一个值来注释文档树的每个元素。属性根据级联和继承部分中描述的机制分配值。
部分值的计算取决于适合目标媒体类型的格式化算法。例如，如果目标媒体是屏幕，则用户代理应用视觉格式模型。

从注释的文档树生成一个 格式化结构。通常，格式化结构与文档树非常相似，但它也可能有很大差异，特别是当作者使用伪元素和生成的内容时。首先，格式化结构不必是“树形”的 - 结构的性质取决于实现。其次，格式化结构可能包含比文档树更多或更少的信息。例如，如果文档树中的某个元素的'display'值为'none ' 属性，该元素将不会在格式化结构中生成任何内容。另一方面，列表元素可以在格式化结构中生成更多信息：列表元素的内容和列表样式信息（例如，项目符号图像）。
请注意，在此阶段，CSS用户代理不会更改文档树。特别是，由样式表生成的内容不会反馈到文档语言处理器（例如，用于重新分析）。

将格式结构转移到目标媒体（例如，打印结果，在屏幕上显示它们，将它们渲染为语音等）。
2.3.1 画布
对于所有媒体，术语画布描述了“格式化结构呈现的空间”。画布对于空间的每个维度都是无限的，但是渲染通常发生在画布的有限区域内，由用户代理根据目标媒体建立。例如，渲染到屏幕的用户代理通常施加最小宽度并基于 视口的尺寸选择初始宽度。用户代理渲染到页面通常会施加宽度和高度限制。听觉用户代理可能对音频空间施加限制，但不及时。

2.3.2 CSS 2.1寻址模型
CSS 2.1 选择器和属性允许样式表引用文档或用户代理的以下部分：

文档树中的元素以及它们之间的某些关系（请参阅选择器一节）。
文档树中元素的属性以及这些属性的值（请参阅关于属性选择器的部分）。
元素内容的某些部分（请参阅：第一行和第一个字母的伪元素）。
文档树的元素处于特定状态时（请参阅伪类的部分）。
文档将在其中呈现的画布的某些方面。
一些系统信息（请参阅用户界面部分）。
2.4 CSS设计原则
本节不具规范性。

CSS 2.1作为CSS2和CSS1之前，基于一组设计原则：

向前和向后兼容性。CSS 2.1用户代理将能够理解CSS1样式表。CSS1用户代理将能够读取CSS 2.1样式表并丢弃他们不了解的部分。此外，没有CSS支持的用户代理将能够显示样式增强的文档。当然，CSS所带来的风格增强功能将不会呈现，但所有内容都将呈现。

对结构化文档的补充。样式表补充结构化文档（例如，HTML和XML应用程序），为标记文本提供样式信息。应该很容易改变样式表，而对标记影响很小或没有影响。

供应商，平台和设备独立性。样式表使文档能够保持供应商，平台和设备独立。样式表本身也是供应商和平台独立的，但CSS 2.1允许您针对一组设备（例如打印机）的样式表。

可维护性。通过指向文档样式表，网站管理员可以简化网站维护并保持整个网站的一致外观。例如，如果组织的背景颜色发生变化，则只需要更改一个文件。

简单。CSS是一种简单的风格语言，它是人类可读和可写的。尽可能最大限度地保持CSS属性彼此独立，通常只有一种方法可以达到某种效果。

网络性能。CSS提供了如何呈现内容的紧凑编码。与图像或音频文件相比，作者经常使用这些文件来实现某些渲染效果，但样式表通常会减少内容大小。而且，需要打开更少的网络连接，这进一步提高了网络性能。

灵活性。CSS可以通过多种方式应用于内容。其关键特性是能够级联默认（用户代理）样式表，用户样式表，链接样式表，文档头以及构成文档主体的元素的属性中指定的样式信息。

丰富。为作者提供丰富的渲染效果，增加了网络作为表达媒介的丰富性。设计师一直渴望在桌面出版和幻灯片放映应用中常见的功能。一些所要求的渲染效果与设备独立性相冲突，但CSS 2.1在向设计者提供他们的请求方面有很长的路要走。

替代语言的绑定。本规范中描述的一组CSS属性为视觉和听觉呈现形成了一致的格式模型。这种格式化模型可以通过CSS语言访问，但也可以绑定到其他语言。例如，JavaScript程序可能会动态地改变某个元素的“颜色”属性的值。

辅助功能。若干CSS功能将使残障用户更容易访问Web：

控制字体外观的属性允许作者消除无法访问的位图文本图像。
定位属性允许作者消除标记技巧（例如，不可见图像）以强制布局。
！重要规则的语义 意味着具有特定演示要求的用户可以覆盖作者的样式表。
所有属性的“继承”值提高了级联的一般性，并允许更容易和更一致的样式调整。
改进的媒体支持（包括媒体组和盲文，浮雕和tty媒体类型）将允许用户和作者为这些设备定制页面。
