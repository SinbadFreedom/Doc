2.搭建Java开发环境

① 下载Java SDK
② 安装
③ windows7配置Java环境变量
④ 重启电脑
⑤ 测试
⑥ 相关文章

SDK全称(Standard Development Kit),标准开发工具包. java和JDK的关系,就像鱼和水的关系.

① 下载Java SDK🔝

1.点击链接:JDK官网下载链接

2.选中Java SE Development Kit 8u144中的Accept License Agreement.

3.选择适合自己操作系统的版本点击链接下载.windows x86是32位版本,windows x64是64位版本.

windows系统位数

如何分辨windows系统是32位还是64位

看我一图:

② 安装🔝

点击下一步直到完成,采用默认安装配置.

③ windows7配置Java环境变量🔝

1.计算机->点右键->属性->高级系统设置. 看我一图:

2.弹出窗口中点击->环境变量. 看我一图:

3.新建用户变量

在用户变量栏中点击->新建按钮.弹出页面变量名一栏中输入:

classpath

变量值一栏中输入:

.;C:\Program Files\Java\jdk1.8.0_144\lib

变量值

由`.;`加`JDK安装目录`加`lib`组成.

Java用户变量classpath配置.看我一图:

4.配置系统变量Path

在系统变量栏中找到Path->点击编辑按钮增加

;C:\Program Files\Java\jdk1.8.0_144\bin

系统变量路径

添加字符串由;加上JDK安装目录加bin目录组成.

看我一图:

  • windows8配置Java环境变量,参考以上配置步骤.
  • windows10配置Java环境变量,参考以上配置步骤.

JDK与JRE

Java默认安装目录C:\Program Files\Java\jdk1.8.0_144. 在jdk1.8.0_144的同级目录有个文件夹jre1.8.0_144,是Java运行环境(Java Runtime Environment),系统运行Java程序使用.

④ 重启电脑🔝

配置完成重启电脑,使环境变量生效.

⑤ 测试🔝

1.打开命令提示符

鼠标点击桌面左下角windows图标->所有程序->附件->命令提示符.

相关链接

如何打开windows命令提示符

2.测试环境变量

输入命令:

java -version

输出java版本号信息说明配置正确.看我一图:

3.测试用户变量

输入命令:

javac

输出javac命令行信息说明配置正确.看我一图:

执行java,javac错误提示

如果执行java命令时在命令行窗口提示错误,表示环境变量配置有问题,需要检查Java环境变量配置.
运行java命令提示'javac'不是内部或外部命令,也不是可运行的程序或批处理文件.检查Java用户变量配置
